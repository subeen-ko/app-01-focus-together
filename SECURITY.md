# Security Policy

## 현재 보안 모델

Workout Timer는 현재 로그인, 서버 API, 데이터베이스 쓰기 기능이 없는 정적 프론트엔드 앱입니다.
따라서 사용자 비밀번호, 결제 정보, 개인 메시지, 비공개 운동 기록을 저장하지 않습니다.

## 배포 전 필수 점검

- `.env` 파일과 비밀키를 Git에 커밋하지 않기
- `VITE_` 환경변수에 비밀키를 넣지 않기
- GitHub Dependabot 보안 업데이트 확인
- Cloudflare Pages 배포 로그에 민감 정보가 출력되지 않는지 확인
- Sentry 같은 모니터링을 붙일 경우 개인정보를 수집하지 않도록 설정
- 추후 로그인, 관리자, 결제, 기록 저장 기능을 추가하면 서버 권한 체크와 입력값 검증을 별도로 설계

## 환경변수 규칙

현재 버전은 필수 환경변수가 없습니다.

`VITE_` 접두사는 비밀 저장소가 아닙니다. 해당 값은 빌드 결과물에 포함되어 브라우저에서 확인될 수 있습니다.
AWS, 문자, 결제 API, 관리자 토큰, 서버 전용 키는 프론트 코드나 `VITE_` 환경변수에 넣지 않습니다.

## 취약점 신고

실제 서비스의 취약점을 발견했다면 공개 Issue에 공격 방법이나 개인정보를 적지 말고, GitHub 저장소의 **Security → Report a vulnerability**를 사용하세요.
