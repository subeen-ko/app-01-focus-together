# 운동 타이머

운동 루틴을 세트 단위로 반복할 수 있는 레트로 스타일 인터벌 타이머입니다.
집중/휴식이라는 초기 컨셉에서 출발했지만, 지금은 **운동 시간과 쉬는 시간을 빠르게 설정하고 세트별로 반복하는 운동 타이머** 방향으로 정리했습니다.

## 주요 기능

- 운동 시간과 쉬는 시간 분리 설정
- `+1초`, `+10초`, `+1분`, `+1시간` 빠른 시간 추가
- 반복 세트 수 설정
- 운동 → 휴식 → 다음 세트 자동 전환
- 마지막 5초만 소리 / 매초마다 소리 모드 선택
- 모바일 브라우저 소리 재생 fallback 적용
- 운동 중 화면이 꺼지지 않도록 Screen Wake Lock 지원
- 모바일에서 긴 시간 표시가 화면을 밀지 않도록 반응형 보정
- 8-bit 레트로 게임 콘솔 느낌의 UI

## 왜 이렇게 바뀌었나

초기 아이디어는 ADHD 사용자를 위한 소셜 집중 타이머였지만, 구현과 테스트를 거치면서 현재 버전은 더 단순하고 즉시 쓸 수 있는 **운동용 인터벌 타이머 MVP**로 방향을 좁혔습니다.

방 만들기, 채팅, 레디고 같은 소셜 기능은 기능 범위가 커지고 보안·실시간 동기화까지 고려해야 해서, 첫 번째 공개 포트폴리오 앱에서는 덜어냈습니다. 대신 실제 모바일에서 바로 쓸 수 있는 타이머 경험에 집중했습니다.

## 배운 점 / 교훈

이번 프로젝트에서 얻은 핵심 교훈은 “앱은 기능보다 실제 사용 환경에서 잘 작동하는지가 더 중요하다”는 점입니다.

1. **컨셉은 만들면서 바뀔 수 있다**
   처음에는 ADHD 집중 앱이었지만, 실제로는 운동 타이머로 쓰기 좋은 구조가 되었습니다. 중요한 건 처음 아이디어를 고집하는 것이 아니라, 현재 앱이 가장 잘 해결하는 문제를 정확히 다시 이름 붙이는 것입니다.

2. **모바일은 데스크톱과 다르게 봐야 한다**
   PC에서는 숫자가 잘 보여도 모바일에서는 `01:00:00` 같은 긴 시간이 화면을 밀 수 있습니다. 그래서 실제 사용 기기 기준으로 레이아웃을 다시 확인해야 합니다.

3. **소리는 “코드가 있다”와 “폰에서 들린다”가 다르다**
   모바일 브라우저는 자동 재생을 강하게 막습니다. 사용자가 직접 누른 버튼 안에서 오디오를 깨우고, WebAudio뿐 아니라 짧은 WAV fallback까지 준비해야 실제로 들릴 수 있었습니다.

4. **작은 문구도 UX다**
   `매초 띠`보다 `매초마다`가 더 자연스럽습니다. 버튼 문구 하나도 사용자가 기능을 이해하는 속도에 영향을 줍니다.

5. **배포 후 피드백이 진짜 개발이다**
   Cloudflare Pages에 올린 뒤 모바일 소리, 화면 유지, 긴 숫자 표시 같은 문제가 보였습니다. 배포는 끝이 아니라 실제 사용을 시작하는 지점입니다.

6. **포트폴리오는 코드만이 아니라 설명도 중요하다**
   README에는 현재 컨셉, 배포 링크, 스크린샷, 배운 점이 있어야 보는 사람이 앱의 의도와 개선 과정을 이해할 수 있습니다.

- Build output directory: `dist`
